# V2 设计文档 — 功能完善

## 变更概述
基于Codex review后的可玩版本，完善对话系统、交易功能、UI交互和历史真实性。

## 变更清单

### 1. 对话系统完善 [V2-1]

**现状**：只能跟老板娘对话，且每次点击返回固定回复。

**目标**：
- 5个NPC均可对话（科内利斯、安娜、亨德里克、玛丽亚、神秘商人）
- 每个NPC每天至少3-4条不同对话
- 对话内容随天数和价格波动变化：
  - Day1-2：兴奋、鼓吹、FOMO
  - Day3-4：狂热、分歧（有人开始担忧）
  - Day5：恐慌、崩溃、互相指责
- 对话可以自然引出交易机会
- 神秘商人每天出现概率不同，且可能提供假情报

**数据结构变更**：
```typescript
// 从简单的一问一答改为对话树
interface DialogueNode {
  id: string;
  npcId: string;
  day: number;        // 哪一天可见
  text: string;       // NPC说的话
  playerOptions: {
    text: string;
    nextNodeId?: string;   // 下一个对话节点
    action?: 'trade' | 'drink' | 'dismiss';  // 触发行为
  }[];
}
```

- 每个人物有头像缩略图 + 名字 + 一句话状态
- 点击人物 → 进入对话
- 神秘商人有"出现/消失"机制（某些天不在）
- Day5所有人都在，气氛紧张

### 3. 面板互斥 + 账簿/物品栏拆分 [V2-3]

**现状**：交易面板和账簿会重叠显示。

**目标**：
- 底部三个按钮互斥：交易市场 | 我的账簿 | 物品栏
- 点击一个自动关闭其他
- 每个面板从底部滑出，占屏幕下半部分

**我的账簿**（Ledger）：
- 现金流状态（当前现金 + 今日盈亏）
- 持仓列表（品种 + 数量 + 买入均价 + 当前市值 + 盈亏%）
- 交易历史（时间线形式，每笔买入/卖出记录）
- 总资产折线图（简单SVG）

**物品栏**（Inventory）：
- 玩家持有的物品卡片
- 每个物品有图标 + 名称 + 数量
- 可交互物品（如啤酒）有"使用"按钮
- 资产变化实时更新（涨绿跌红闪烁）

### 4. 卖出功能 + 数量校验 + 全仓 [V2-4]

**现状**：只能买入，不能卖出。

**目标**：
- 交易面板增加"买入/卖出"切换标签
- 买入时：
  - 数量选项根据现金自动过滤（买不起的灰色禁用）
  - 新增"全仓"按钮：以当前现金买入最大数量
  - 显示预计花费和剩余现金
- 卖出时：
  - 只能卖出持仓中有的品种
  - 数量选项根据持仓自动过滤
  - 新增"全部卖出"按钮
  - 显示预计收入
- 交易前校验：
  - 买入：现金 >= 总价
  - 卖出：持仓 >= 卖出数量

### 5. 啤酒物品 + 初始持仓 [V2-5]

**现状**：玩家初始只有10000盾现金，没有任何现货。

**目标**：
- 初始持仓调整（基于历史调研结果 V2-0）：
  - 玩家身份是花商，应该持有一些郁金香现货
  - 初始资产：现金 + 若干郁金香球茎 + 少量啤酒
- 新增啤酒物品：
  - 啤酒可在酒馆向玛丽亚购买（3盾/杯）
  - 饮用效果：对话选项解锁（喝了酒NPC更愿意透露信息）
  - 需要生成啤酒图片（木制酒杯，油画风格）
  - 物品栏中显示，可点击"饮用"
- 房产契约和航海股份也需要可以通过交易获得

### 6. 历史调研 [V2-0]

**目标**：调研1637年2月3-5日的真实历史数据，包括：
- 各品种郁金香的真实价格范围（当时用什么货币？guilders）
- 阿姆斯特丹运河房产的真实价格
- 荷兰东印度公司(VOC)股票的价格
- 一个普通花商大概的资产规模
- 泡沫破裂时的具体跌幅

**用途**：
- 调整游戏中的初始价格、波动幅度，使更有历史真实感
- 确定玩家的初始持仓（花商应该持有什么、值多少）
- 结局页面的"历史上的今天"文案

### 7. 小细节优化
- 交易市场中的数量选择按钮：根据当前资金动态计算可选数量
- 价格看板上涨跌幅显示更醒目
- NPC头像在对话时有呼吸动画
